# 医院智能排班与值班管理系统 - 用户指南

## 概览
本系统旨在帮助医院高效管理医生排班、值班及请假调班。系统包含三个主要部分：
1. **后端服务器 (Backend Server)**: 处理核心逻辑、数据库交互及API接口。
2. **管理后台 (Admin Dashboard)**: 供管理人员使用，用于管理医生信息、科室架构及生成排班表。
3. **移动端应用 (Mobile App)**: 供医生使用，用于查看个人排班及发起换班请求。

## 快速上手

### 1. 后端服务 (Backend)
后端服务默认运行在 `http://127.0.0.1:8000`。
- **接口文档**: [Swagger UI](http://127.0.0.1:8000/docs)
- **健康检查**: [数据库状态](http://127.0.0.1:8000/health/db)

### 2. 管理后台 (Admin Dashboard)
管理后台通常运行在 `http://localhost:5173`。
- **登录**: 使用管理员账号登录 (例如: `admin` / `admin123`)。
- **主要模块**: 医生管理、排班生成、统计报表、审计日志等。

### 3. 移动端应用 (Mobile App)
移动端应用运行在 `http://localhost:5175`。
- **登录**: 使用医生账号登录。
- **主要模块**: 我的排班、意愿申报、换班申请、反馈。

## 核心功能指南 (Core Features)

### 1. 移动端 (Mobile App)
供医生使用，提供以下核心功能：
- **排班查看**: 首页Dashboard展示近期排班。
- **意愿申报 (Preference)**: 
    - 每月或每周提交排班意愿（如期望休息日、避免连班）。
    - 入口：首页 -> "意愿申报"。
- **换班调班 (Shift Trade)**:
    - 对已发布排班发起换班申请，选择目标医生和原因。
    - 入口：点击具体班次 -> "换班"。
    - 查看请求：首页 -> "换班请求"。
- **问题反馈 (Feedback)**:
    - 提交Bug、建议或评价。
    - 入口：页面右下角悬浮按钮。

### 2. 管理后台 (Admin Dashboard)
供管理员使用，提供全面的系统管控：
- **资源管理**:
    - **医生管理**: 支持 CRUD 及 **标签管理 (Tags)**（如：资深、孕期、进修等）。
    - **科室/诊室/班次**: 基础数据配置。
- **智能排班**:
    - **生成排班**: 基于规则引擎自动生成。
    - **冲突检测**: 自动检测人员、时间冲突。
- **流程审批**:
    - **换班审批**: 处理医生提交的换班请求。
- **系统监控**:
    - **统计报表**: 查看科室/医生工作量统计。
    - **审计日志**: 追踪系统关键操作（排班修改、删除、审批）。
    - **用户反馈**: 查看并处理移动端提交的反馈。
- **系统设置**:
    - 支持中英文切换 (i18n)。

## 常用启动命令
- **启动后端**: `.\run_backend.ps1`
- **启动管理后台**: `cd frontend-admin && npm run dev`
- **启动移动端**: `cd mobile-app && npm run dev:h5`
